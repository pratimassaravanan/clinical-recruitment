Why this benchmark is different

Workflow-shaped, not toy-shaped.

The environment is built around a patient funnel and operational constraints rather than frame-level rewards. Decisions echo forward through time, which makes interface correctness as important as reward design.

05 / 09
Repo baselines

Four trainable baselines, one shared pure-NumPy actor-critic backbone.

HCAPO
Hierarchical subgoals + hindsight relabeling

Planner / executor separation and relabeled credit assignment over long trajectories.

MiRA
Milestone-aware potential shaping

A learned milestone critic perturbs reward flow without changing the benchmark interface.

KLong
Multi-scale temporal aggregation

Overlapping trajectory segments and TD(lambda)-style credit propagation.

MemexRL
Episodic memory write / read behavior

Learns what to store, what to retrieve, and when memory should influence action choice.

The current documentation treats these as repo baselines. The corrected paper and docs avoid presenting them as externally validated reproductions of named methods.
