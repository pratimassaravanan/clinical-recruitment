## Environment Design

### Domain
Cloud autoscaling — a genuine real-world task (not games/toys). Agent manages VM instances to balance cost, SLA compliance, and resilience under dynamic traffic.

### Episode Structure
- 720 steps = 24-hour simulation (2-minute intervals)
- Diurnal traffic pattern with math.exp modeling
- Pre-computed deterministic traces (reproducible with fixed seeds)

### Tasks (Easy → Medium → Hard)
1. **easy_bench** — Stable diurnal load only. No failures, no spikes. Agent learns basic scaling rhythm.
2. **medium_bench** — Diurnal + traffic bursts + node failures. Tests reactive scaling.
3. **hard_bench** — Full chaos: noisy load, cascading failures, price/carbon spikes, curriculum learning injections.

### Action Space
- `scale_up` (num_instances: 0-20)
- `scale_down` (num_instances: 0-20)
- `do_nothing`
- `migrate_load`

### Observation Space (Pydantic typed)
- timestamp, current_load, cpu_util, memory_util, instance_count
- cost_so_far, sla_violation_minutes, recent_events
- difficulty_level, pending_requests, avg_response_time_ms

### Reward Function (6 components)
1. **SLA compliance** — penalty for SLA violations
2. **Cost efficiency** — reward for keeping costs low
3. **Burst response** — reward for quick reaction to traffic spikes
4. **Failure survival** — reward for handling node failures gracefully
5. **Curriculum response** — reward for adapting to injected challenges (hard only)
6. **Stability** — penalty for thrashing (frequent scale up/down)

### Grader Cost Thresholds (tuned for 720-step episodes)
- Easy: $400-700
- Medium: $500-900
- Hard: $550-1000

### Baseline Scores
- Easy: 0.82
- Medium: 0.68
- Hard: 0.76

### Novelty
Curriculum learning injections — the environment injects new challenge patterns mid-episode that the agent hasn't seen before, testing generalization. Plus cascading failures and price/carbon spikes.

### Environment Variables

```python
MODEL_NAME = os.getenv("MODEL_NAME", "gpt-4o-mini")                    # has default

```

### OpenAI Client
```python
from openai import OpenAI
client = OpenAI(base_url=API_BASE_URL, api_key=HF_TOKEN)
```

### Structured Logging (plain text, matches latest sample format)
```python
log_start(task=task_id, env="vitascale", model=MODEL_NAME)
# → [START] task=<task_id> env=vitascale model=<model>

log_step(step=N, action="scale_up", reward=0.5, done=False, error=None)
# → [STEP] step=<n> action=<action> reward=<reward> done=<true|false> error=<value|null>

log_end(success=True, steps=720, rewards=[...])
# → [END] success=<true|false> steps=<n> rewards=<r1,r2,...>
```

### Score Calculation
```python
score = sum(rewards) / MAX_TOTAL_REWARD  # MAX_TOTAL_REWARD = 720.0
score = min(max(score, 0.0), 1.0)        # clamp [0, 1]
success = score >= SUCCESS_SCORE_THRESHOLD  # threshold = 0.5
```

### Agent Policy
- LLM called every 5 steps (LLM_CALL_INTERVAL=5)
- Rule-based fallback between LLM calls (capacity-based scaling)
- Rule-based fallback on LLM error

## Key Learnings / Gotchas

### HF Spaces Deployment
- Space uses Docker SDK, port 7860
- README.md YAML frontmatter format matters — wrong format causes CONFIG_ERROR
- Must use `app_port: 7860` in frontmatter

### Git & Push
- HF push needs token in URL (strip after push for security)
- When pushing to both remotes, GitHub may reject if behind — use `git pull --rebase` first
- Force push to HF is safe since it's our deployment target

### openenv validate
- Requires: pyproject.toml, uv.lock, server/app.py, openenv-core dep, [project.scripts]
- The `build-backend` must be `setuptools.build_meta` (NOT `_legacy:_Backend`)
- `uv lock` generates uv.lock from pyproject.toml

### inference.py Format
- **Critical:** "Any deviation in field names, ordering, or formatting will result in incorrect evaluation scoring"
- Must use `log_start/log_step/log_end` functions with exact field names from sample
- Plain-text stdout lines, not JSON
- `[END]` line must **not** include `score=` in the latest guideline version
- All LLM calls MUST use `from openai import OpenAI`
- HF_TOKEN must have NO default value
- API_BASE_URL and MODEL_NAME MUST have defaults

### Grader Tuning
- Initial cost thresholds ($35-80) were unrealistic for 720-step episodes
- Tuned to $400-1000 range for realistic cloud costs
- Graders must be deterministic and return 0.0-1.0

### Infra Constraints
- Runtime of inference < 20 min
- Must run on vcpu=2, memory=8gb
- LLM_CALL_INTERVAL=5 helps stay within time/cost budget
